Execute one step at a time. Report test results before moving to the next step.

**Why:** User prefers incremental progress with verification at each stage rather than bulk implementation.

**How to apply:** When implementing multi-step work items, complete one step, run relevant tests, report results, then proceed to the next step only after confirming the previous one passes.
